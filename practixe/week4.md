# 4주차 실습 정리
## 오늘 한 것
- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정 차이 관찰
## 관찰 내용
### 원형 vs AABB
원형은 물체가 회전해도 히트박스의 크기나 방향이 달라지지 않는다.
하지만 AABB는 물체가 회전할 경우 히트박스의 크기가 달라지고 이로인해 실제 크기보다 더욱 큰 히트박스를 갖게 되어
눈으로 봤을 때는 충돌하지 않았지만 충돌한 판정이 일어나게 되는 단점을 갖고 있음.
### AABB vs OBB
AABB는 물체가 회전할때 히트박스가 달라지고 보이는것보다 히트박스가 크게 설정될 수 있기에 충돌 판정이 보이는것과 다르게 작동할 수 있음
하지만 OBB는 물체의 크기를 따라 히트박스가 설정되고 정해진 히트박스가 같이 회전하여 히트박스가 달라지지 않아 더욱 더 정확한 충돌판정을 갖게 할 수 있다. 그러나 OBB는 AABB보다 복잡한 구성을 갖고있다.
## AI와의 대화에서 배운 것
원형 충돌 감지 코드를 작성하기 위해 sqrt를 쓰는이유를 묻고 없이 구현하는 방법을 물었을 때, 수학적원리에 대해 먼저 개념을 설명하고 sqrt함수 없이 코드를 작성하는 방법을 설명해주어 sqrt함수가 하는 일이 무엇이고 어떤 구조로 작동하는 지 알 수 있었던 내용이 기억에 남는다.
## 내 게임에 적용한다면
공격스킬의 이펙트와 개체의 충돌판정시 데미지를 입게하거나, 벽에 플레이어가 충돌할 경우 벽 방향으로 이동하지 못하게 하는 등 여러가지 접촉관련 코드를 짤때 사용될 것 같다.
